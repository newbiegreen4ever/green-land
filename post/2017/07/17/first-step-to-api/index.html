Keywords = {
  "Open Authorization" : "Aka OAuth, client can get a key from the server by simple steps",
  "The User" : "A person who wants to connect two websites they use",
  "The Client" : ["The website that will be granted access to the user's data", "e.g. Codepen.io"],
  "The Server" : ["The website that has the user's data", "e.g. GitHub.com"],
  "Access Token" : "A secret that the client obtains upon successfully completing the OAuth process which enables the client to fetch data from server",
  "Scope" : "To what extent the client can access the user's data"
}
Step OAuth 1 OAuth 2
1 User Tells Client to Connect to Server User Tells Client to Connect to Server
** Client to ask the server for a request token (No such step)
** User digitally signs the request token (No such step)
2 Client Directs User to Server Client Directs User to Server
Server sends back Client by callback URL once User authenticates Server sends back Client by callback URL once User authenticates
3 User Logs-in with username and password to Server and Grants Client Access User Logs-in with username and password to Server and Grants Client Access
4 Server Sends User Back to Client by Callback URL, Along with Authorization Code Server Sends User Back to Client by Callback URL, Along with Authorization Code
5 Client Exchanges Authorization Code together with Client Secret Key for Access Token Client Exchanges Authorization Code together with Client Secret Key for Access Token
6 Client Fetches Data from Server Client Fetches Data from Server
** (No such thing) Client Fetches Data under LIMITED OAuth Scope
** (No such thing) Access tokens expire
