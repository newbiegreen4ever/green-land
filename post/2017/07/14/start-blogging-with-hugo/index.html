Why blogging?

There are several good reasons to blog.

| Reasons behind blogging | Explanation

:: | :————————— | :—————————————————————————————————————————————————– 1. | Build memory palace | A personal blog enables you to organize content in a fully customized fashion. This makes it easier to collect and retrieve information for later use. 2. | Express oneself with clarity | Writing is the best way to capture diffusing thoughts and fleeting inspirations. 3. | Play with self-host | Self-hosting sounds cool.

I. Get started with Jekyll

A friend mentioned Jekyll when talking about building and hosting sites. Being naively stupid, I started googling and reading about Jekyll right away without giving much thought on other alternatives. This mistake cost me nearly 8 hours of leisure time.

The official Jekyll on Windows walkthrough is my main source of information. I installed Jekyll via Chocolatey, a package manager on Windows.

II. Pack my Ruby

Before I could run Jekyll on Window, I had to install Ruby, the engine that runs Jekyll. According to the official Jekyll on Windows guide, there are 4 steps:

  1. Install Ruby and Ruby development kit
  2. Configure Ruby development kit
  3. Install Nokogiri gem
  4. Install GitHub-pages gem

III. Encounter Gem roadblock

Gem is a type of package written in Ruby, and it is notoriously troublesome to install and run on Windows (incompatible people just can’t get along). I could perform steps 1 to 3, but the installation of GitHub-pages gem failed due to dependencies issue.

Somehow I got stuck in the installation of GitHub-pages. I could not run gem install bundle. After inserting this line gem 'wdm', '>= 0.1.0' if Gem.win_platform? in GemFile, it still couldn’t run

I nearly gave up on setting my site. As Jekyll is the de facto standard static site generator, I thought if I could not use Jekyll properly, I would never be able to host a site.

III. Go again with Hugo

Later I ran across Hugo. It is a fast static site generator that can run without dependencies. I decided to give it a try.

With Hugo, a decent site could be generated within seconds effortlessly. This blog adopts Xmin theme which offers nice content organization. Apart from the blinding white color scheme, I have no further comment on it as a happy user.

IV. Run with GitHub page

“How to get your website online easily with Github” written by Julio F. has been a great help. I knew that decent developers should dig deep into official GitHub Pages documentations, but GUI is much more friendly to newbie. I wasted > 10 hours bumping my head against the wall trying to push my site via command line.

My lesson in this painful deployment:

Anything that works is a solution, regardless of being stupid or not.

Finally….

If I want to refresh my blog, simply do the following

Method 1 : Grow a new branch

  1. delete the public directory inside green-land
  2. rerun hugo --buildDrafts --theme=hugo-xmin without server
  3. upload everything in the newly generated public to GitHub repository in a new branch
  4. delete the existing master branch
  5. rename the new branch as master

Method 2 : Update master branch

  1. delete the public directory inside green-land
  2. rerun hugo --buildDrafts --theme=hugo-xmin without server
  3. upload everything in the newly generated public to GitHub repository in a new branch
  4. merge the branches
Q: What if nothing shows up at GitHub-Page?
A: If nothing shows at the GitHub-Page address, I should always check whether the site's config.toml has been properly configured. If there is one more /, the site URL would be https://username.github.io/sitename/sitename/ , rather than https://username.github.io/sitename/ specified in GitHub repository setting.

P.S. Still a long way to go

This post ends here, but my journey with Hugo will continue. I understand that I cannot reach every place at once, and issues have to be solved one-by-one. This is my list of issues to be solved.

  1. Use HTTPS connection : It turns out HTTPS is used by default
  2. Use my custom domain
  3. Customize Hugo theme
  4. Deploy by Netlify

Summary

  • In retrospect, I realize that it only took me little effort to build a Hugo site. Hugo is newbie-friendly enough.
  • Making endless theme-twitching attempts is such a time sink! I should avoid that next time! If I can’t fix it, leave it alone! Don’t blindly bump my head against the wall.
  • Recapturing what, how and when things are done is a good habit.
